SDG 6 — Clean Water and Sanitation

Access to clean water is a fundamental human right, yet millions still lack safe drinking water and adequate sanitation. This SDG drives my motivation to build apps that empower communities to monitor and protect their water resources in real time.

1

Lab 1 — UI Design

Jetpack Compose Layouts

Reflection

Lab 1 introduced me to Jetpack Compose and the declarative approach to building Android UIs. I learned how to structure composables, apply modifiers, and organise content using Column, Row, and Box layouts. This foundational experience shifted my thinking from traditional XML-based design to a more dynamic, code-first UI paradigm.

2

Lab 2 — Interaction

State Management & Events

Reflection

In Lab 2, I explored state management in Compose using remember and mutableStateOf, allowing UI components to react to user input in real time. Handling click events and conditional rendering helped me understand how Compose recomposes only the parts of the UI that change. This lab made me appreciate how much cleaner reactive programming is compared to imperative view updates.

3

Lab 3 — Material Design

Material 3 Components & Theming

Reflection

Lab 3 walked me through Material 3 design guidelines, including dynamic colour theming, typography scale, and component styles like Cards, Buttons, and TopAppBar. I learned how to apply a consistent visual language across the app while keeping it adaptable to different device themes. Seeing how small theming decisions can drastically improve the perceived quality of an app was a major takeaway.

4

Lab 4 — Navigation

Jetpack Navigation Component

Reflection

Lab 4 introduced multi-screen navigation using Jetpack's NavController and NavHost. I implemented a bottom navigation bar that lets users move seamlessly between different sections of the app while preserving back-stack behaviour. Understanding how to pass arguments between composable destinations and manage the navigation graph prepared me well for building more complex, production-ready apps.

5

Lab 5 — Room Database

Local Persistence with Room

Reflection

Lab 5 brought everything together through Room, Android's abstraction layer over SQLite. I defined entities, DAOs, and a database class, then connected them to a ViewModel using Flow so the UI automatically updates when data changes. This lab gave me confidence in handling offline-first data storage, which is essential for any mobile app that needs to work reliably without a constant internet connection.

Capstone Work

Projects

Project 1 — Lab 4 + Working Features

AirKu — Infrastructure Reporter

Problem Statement

Citizens lack a real-time, evidence-based method to report infrastructure issues like pipe leaks to local authorities. Delayed reporting leads to water wastage and prolonged disruption for communities.

Lessons Learned

Building Project 1 pushed me to integrate multiple screens with a coherent navigation flow while managing UI state across them. I learned how to design user-friendly forms with input validation and how bottom navigation can greatly improve the discoverability of app features. Most importantly, I realised that a clear user journey — from spotting a problem to successfully submitting a report — requires careful UX planning, not just functional code.

Project 2 — Lab 5 + API + Sensors + Cloud

AirKu — Smart Water Monitoring System

Problem Statement

Citizens lack a real-time, evidence-based method to report infrastructure issues like pipe leaks to local authorities. Without a smart, sensor-integrated platform, authorities cannot respond swiftly or track recurring problem zones effectively.

Lessons Learned

Project 2 was the most challenging and rewarding milestone of the course. Integrating Room for offline caching, Retrofit for API calls, device sensors for geo-tagged reports, and Firebase Firestore for cloud sync taught me how these layers interact and how to handle failure states gracefully. I also gained practical experience with coroutines and ViewModels in a multi-feature context, which solidified my understanding of clean architecture in Android development.

Looking Back

Reflection & Learning Journey

My journey through this course began with the basics of Jetpack Compose in Lab 1 and has grown into building full-stack Android applications by Project 2. Each lab built on the previous one — learning UI fundamentals, then state management, then Material Design principles, then navigation, and finally data persistence — creating a structured ladder of skills that I could apply directly to real projects. The experience of turning a simple water-reporting concept into a working app with offline support, API integration, and cloud sync has shown me how much can be achieved with the right tools and a clear problem to solve. Moving forward, I am excited to deepen my knowledge of clean architecture patterns, explore Kotlin Multiplatform, and continue building apps that contribute meaningfully to societal challenges like those addressed by SDG 6.
